append

return type jQuery collection
syntax .geomap( "append" , Object shape ( GeoJSON object ) [ , Object style ( geomap style ) ] )
usage 
$("map").geomap( "append", { type: "Point", coordinates: [ -71, 40 ] } )
$("map").geomap( "append", { type: "Point", coordinates: [ -71, 40 ] }, { stroke: "#11117f", strokeWidth: "3px" } )

The append method adds a shape to the map and then redraws the map. In this documentation the word shape means a GeoJSON geometry object, GeoJSON feature or GeoJSON feature collection. Each feature in a FeatureCollection's features property is added as a separate shape whereas the other collection geometry types, e.g., MultiPoint, are added as a single shape. This is important distinction when considering the find method in that find can potentially return only one shape of a FeatureCollection but will return all shapes in a MultiPoint if only one intersects the find position.

The geomap widget maintains a reference to your shape. It will not change the object in any way. You can use the same object in calls to remove to remove the shape from the map at any time. You can change the geometry of the original object at any time by updating the coordinates property and geomap will draw the new shape when it refreshes. Changing the type of geometry, e.g., from Point to LineString, is not recommended and geomap's behavior is currently undefined. If you must change the shape type, you should first call remove on the original object and append on a new one.

If $.geo.proj is not null, all coordinates of all geometry (whether plain or part of a feature) must be in geodetic coordinates, e.g., for a Point object coordinates[0] is longitude and coordinates[1] is latitude. If $.geo.proj is null, all coordinates must be in the widget's internal projection as defined by tilingScheme for tiled map services or bboxMax for shingled (dynamic) map services.

The optional style argument modifies how geomap draws the specific geometry or feature you are adding. Properties supplied in this style will override ones of the same name in geomap's base shapeStyle. Properties not referenced are inheritied from the base style and can change with future calls to shapeStyle. Please see the shapeStyle method documentation for information about what properties are valid for this object.

The jQuery UI widget factory returns the original jQuery collection to maintain call chaining.
